機能

3つのコア機能

💬

自然言語クエリ

「進捗は？」「リスクは？」と聞くだけ。AIがプロジェクトデータを参照し、ロールに合った言葉で回答。

🔄

言語ブリッジ

技術用語 ⟷ ビジネス用語 ⟷ デザイン用語を自動変換。「PRレビュー待ち」→「コード確認の承認待ち」→「ハンドオフ待ち」のように翻訳。

🤝

ニーズ分析

経営層・エンジニア・デザイナーの三者間で互いに何を求めているかをAIが自動分析。コミュニケーションギャップを可視化。

API

Bridge API エンドポイント

POST /query

自然言語でプロジェクトについて質問。role パラメータで経営語/技術語を切替。

GET /metrics/{project}

Issue完了率、PR状況、スプリント進捗、デプロイ状況、ブロッカー一覧を取得。

GET /summary/{project}

プロジェクトの包括的サマリーを生成。?role=management でロール指定。

GET /needs/{project}

三者のニーズをAIが分析。相互理解のギャップを特定。

GET /figma/{file_key}

Figmaファイルのコンポーネント・コメント・バージョン情報を取得。

効果

導入メリット

  • 1 意思決定の加速 — 技術報告を待たず、経営層がリアルタイムでプロジェクト状況を把握。判断に必要な情報がビジネス語で即座に届く。
  • 2 報告コストの削減 — エンジニアが経営向け報告書を手作業で作る時間をゼロに。AIがデータから自動生成。
  • 3 ミスコミュニケーション防止 — 「技術的負債」「MTTR」などの専門用語を相手が理解できる言葉に自動変換。認識のズレを解消。
  • 4 ニーズの見える化 — お互いが何を求めているか、データに基づいて自動分析。会議前に論点が明確になる。
  • 5 既存インフラに統合 — Jira / GitHub / DevLake の既存データを活用。新しいツール学習不要。チャットで聞くだけ。
