API

Bridge API エンドポイント

POST /query

自然言語でプロジェクトについて質問。role パラメータで経営語/技術語を切替。

GET /metrics/{project}

Issue完了率、PR状況、スプリント進捗、デプロイ状況、ブロッカー一覧を取得。

GET /summary/{project}

プロジェクトの包括的サマリーを生成。?role=management でロール指定。

GET /needs/{project}

三者のニーズをAIが分析。相互理解のギャップを特定。

GET /figma/{file_key}

Figmaファイルのコンポーネント・コメント・バージョン情報を取得。

効果

導入メリット

  • 1 意思決定の加速 — 技術報告・デザイン承認を待たず、経営層がリアルタイムでプロジェクト状況を把握。判断に必要な情報がビジネス語で即座に届く。デザイナーも実装状況をデザイン語で確認でき、承認フローが高速化。
  • 2 報告コストの削減 — エンジニアが経営向け報告書を手作業で作る時間をゼロに。デザイナーのハンドオフ報告（実装差分・コンポーネント進捗）もAIがデータから自動生成。三者間の報告がすべて自動化。
  • 3 ミスコミュニケーション防止 — 技術用語「技術的負債」「MTTR」をビジネス語に、デザイン用語「コンポーネント」→「画面パーツ」、「トークン」→「デザイン規格」のように三者間の専門用語を相手が理解できる言葉に自動変換。認識のズレを解消。
  • 4 ニーズの見える化 — 経営層・エンジニア・デザイナーの三者がお互いに何を求めているか、データに基づいて自動分析。会議前に論点が明確になる。
  • 5 既存インフラに統合 — Jira / GitHub / DevLake / Figma の既存データを活用。新しいツール学習不要。チャットで聞くだけ。
